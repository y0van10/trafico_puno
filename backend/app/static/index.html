Lógica del Clasificador XGBoost

El estimador predictivo de XGBoost (eXtreme Gradient Boosting) utiliza una infraestructura lógica basada en Árboles de Decisión Enlazados.

En lugar de computar una sola regla masiva, XGBoost entrena múltiples árboles ligeros secuenciales donde cada nuevo estimador se enfoca en afinar los errores de predicción del estimador anterior.

Estructura del Árbol de Clasificación Local: 
   [¿Es Hora Pico?] 
     ├── Sí ──> [¿Clima Adverso?]
     │            ├── Sí ──> Tránsito Pesado (Congestión Alta)
     │            └── No ──> Tránsito Moderado (Congestión Media)
     └── No ──> [¿Zona Comercial?]
                  ├── Sí ──> Tránsito Moderado (Congestión Media)
                  └── No ──> Tránsito Fluido (Congestión Baja)
Este flujo condicional se computa asíncronamente en milisegundos en el host del backend.

Mantenimiento y Retraining Loop

Los patrones de movilidad urbana puneña varían con el tiempo (Model Drift). Nuestro script `retrain.py` automatiza la calibración del modelo:

  • Ingesta de Datos Recientes: Carga nuevos registros del flujo vial para actualizar el dataset.
  • Búsqueda por Rejilla (Grid Search): Calibra **108 combinaciones** de hiperparámetros de XGBoost.
  • Filtro de Desempeño: Compara el nuevo coeficiente $R^2$. Si la precisión disminuye más de un 5%, el pipeline rechaza la promoción para resguardar la API activa.
